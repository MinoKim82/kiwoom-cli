# Kiwoom-CLI

이 프로젝트는 **Kiwoom CLI** 프로그램의 설정 방법, 기본 사용법 및 주요 비즈니스 시나리오별 활용 가이드를 제공합니다.

---

## 1. 개요 (Overview)

Kiwoom CLI는 키움증권 REST API를 이용하여 각 계좌별 자격 증명을 관리하고, 보유 종목 및 평가 잔고 데이터를 조회하는 파이썬 기반 명령줄 인터페이스(CLI) 도구입니다.

### 주요 기능
- **다중 계좌(Multi-account) 컨텍스트 분리**: 각 계좌별로 발급되는 고유 API 자격 증명(`config.json`)과 세션 토큰(`tokens.json`)의 독립적 저장 및 관리
- **세션 토큰 자동 갱신 (Auto Token Refresh)**: API 요청 시 토큰의 KST 만료 임박(10분 이내) 여부를 계산하여 자동으로 토큰 재발급 및 갱신 수행
- **연속 조회 처리 (Pagination)**: 보유 종목이 많은 계좌의 경우, API의 연속 조회(`cont-yn: "Y"`) 루프를 자동으로 돌려 하나의 리스트로 결합
- **가독성 높은 터미널 출력**: 표 형식으로 정돈된 계좌 정보 및 보유 종목 데이터 출력 (천 단위 콤마 포맷팅, 수익률 부호 처리 포함)

## 2. 설치 및 환경 설정 (Installation & Setup)

### 요구사항
- Python 3.8 이상
- 관련 종속성 라이브러리: `requests`, `click`

### 설치 방법

#### 1) GitHub 저장소 클론 및 디렉토리 이동
```bash
git clone https://github.com/MinoKim82/stock-kiwoom-cli.git
cd stock-kiwoom-cli
```

#### 2) 패키지 로컬 설치
다음 명령어를 실행하여 로컬 환경에 패키지를 설치합니다. 설치 후에는 `kiwoom` 명령어를 터미널에서 바로 사용할 수 있습니다.

```bash
# 로컬 개발/실행 모드 설치
pip install -e .

# 테스트 및 개발용 의존성(pytest, requests-mock)을 포함하여 설치
pip install -e .[dev]
```

### 환경 설정 파일 생성
계좌별 API Key와 Secret Key 데이터를 담고 있는 `config.json` 파일을 홈 디렉토리 내 `.kiwoom` 폴더에 생성해야 합니다.

1. 터미널에서 홈 디렉토리에 폴더 생성:
   ```bash
   mkdir -p ~/.kiwoom
   ```
2. `~/.kiwoom/config.json` 파일을 작성합니다. 계좌번호를 키로 하여 각 계좌별 정보를 등록합니다.
   ```json
   {
     "accounts": {
       "1234567890": {
         "appkey": "appkey_value_for_account1",
         "secretkey": "secretkey_value_for_account1"
       },
       "9876543210": {
         "appkey": "appkey_value_for_account2",
         "secretkey": "secretkey_value_for_account2"
       }
     }
   }
   ```

## 3. CLI 명령어 레퍼런스 (CLI Reference)

모든 명령어는 기본적으로 `--account` (또는 `-a`) 옵션을 필수로 요구합니다.

### 3.1. 평가 잔고 및 보유 종목 조회 (`balance`)
계좌의 예수금, 총 매입금액, 평가손익 및 개별 보유 종목의 상세 내역을 표 형태로 출력합니다.

```bash
kiwoom --account 1234567890 balance
```
* **출력 예시**:
  ```text
  ==================================================
    [1234567890] 계좌 평가 현황
  ==================================================
  계좌 번호   : 1234567890
  추정예탁자산 : 15,230,000 원
  총 매입금액  : 10,000,000 원
  총 평가금액  : 12,500,000 원
  총 평가손익  : +2,500,000 원
  총 수익률    : +25.00%
  ==================================================

  ==========================================================================================
    [1234567890] 보유 종목 현황
  ==========================================================================================
  종목코드     | 종목명             | 보유수량     | 매입단가     | 현재가      | 평가손익     | 수익률

  005930   | 삼성전자           | 100 주    | 70,000 원 | 75,000 원 |   +500,000 |  +7.14%
  000660   | SK하이닉스          | 50 주     | 120,000 원 | 140,000 원 |  +1,000,000 | +16.67%
  ==========================================================================================
  ```

## 4. 실무 사용 시나리오 (Scenarios)

### 시나리오 A: 다중 계좌의 자산 포트폴리오를 빠르게 순차 조회하기
다중 계좌를 보유한 사용자가 각 계좌의 자산 평가 상태를 한 번에 검토해야 할 때 유용합니다.

1. **상황**: `1234567890`, `9876543210` 두 계좌의 보유 포트폴리오를 차례대로 모니터링하고자 합니다.
2. **명령어 실행**:
   ```bash
   # 첫 번째 계좌의 자산 잔고 확인
   kiwoom -a 1234567890 balance
   
   # 두 번째 계좌의 자산 잔고 확인
   kiwoom -a 9876543210 balance
   ```
3. **내부 동작**:
   - CLI 실행 시 `--account` 매개변수에 전달된 값에 따라 `~/.kiwoom/tokens.json`에서 계좌별 세션 토큰 정보를 개별 매핑합니다.
   - 따라서 서로 다른 계좌의 토큰이 혼선 없이 깔끔하게 분리되어 조회됩니다.

### 시나리오 B: 만료 10분 전 토큰의 완전 자동 갱신
토큰이 만료되어 API 호출이 실패하는 상황을 미연에 방지합니다.

1. **상황**: `1234567890` 계좌가 어제 발급받은 세션 토큰이 캐시되어 있으며, 현재 만료 시점까지 약 5분이 남았습니다.
2. **명령어 실행**:
   ```bash
   kiwoom -a 1234567890 balance
   ```
3. **내부 동작**:
   - `KiwoomClient` 내부의 `get_valid_token()` 함수가 호출됩니다.
   - `tokens.json`에 기록된 토큰의 `expires_dt`를 읽고 만료 시간을 파싱합니다.
   - `expires_at - datetime.now() < 10분` 조건을 만족(만료 10분 전 상태)하므로 `refresh_needed`가 `True`로 전환됩니다.
   - 프로그램이 백그라운드에서 즉시 `oauth2/token` API를 재호출하여 새 토큰을 발행받고, 로컬 `tokens.json`을 갱신합니다.
   - 사용자는 별도의 경고나 오류 없이 갱신된 새로운 토큰으로 정상 조회를 완수하게 됩니다.

### 시나리오 C: 20개 이상의 많은 종목을 보유한 계좌 조회 (연속 조회)
키움 API는 한 번의 요청에 반환할 수 있는 종목 수에 제한이 있습니다. 본 프로그램은 이를 페이징 처리하여 무한 루프로 안전하게 수집합니다.

1. **상황**: `9876543210` 계좌에 코스피/코스닥 종목이 총 45종목 등록되어 있습니다.
2. **명령어 실행**:
   ```bash
   kiwoom -a 9876543210 balance
   ```
3. **내부 동작**:
   - API 서버에 첫 페이지 데이터를 요청합니다.
   - 응답 헤더의 `cont-yn`이 `"Y"`이고 `next-key`에 다음 페이지 조회를 위한 식별키가 존재하면 루프가 유지됩니다.
   - 프로그램이 두 번째 페이지 및 세 번째 페이지를 `next-key` 정보를 담아 연속해서 호출합니다.
   - 수집된 `acnt_evlt_remn_indv_tot` 종목 리스트 45개가 메모리상에서 하나로 병합되어, 단 한 번의 깔끔한 테이블 레이아웃으로 출력됩니다.
